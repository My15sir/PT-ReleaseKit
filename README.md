# PT-BDtool

PT-BDtool 是给 PT 用户整理发种素材用的小工具。  
它会在 VPS 上扫描视频、音频、Blu-ray 原盘目录 `BDMV`、Blu-ray 镜像 `ISO`，自动生成截图、媒体信息，并把结果下载回你的电脑。

## 普通用户先看这个

**如果你只是想用，不要下载这个仓库源码。**

你真正应该拿到的是下面这两种成品之一：

- **Windows**：`PT-BDtool.exe` 或 `PT-BDtool-windows-portable.zip`
- **macOS**：`PT-BDtool.app` 或 `PT-BDtool-macos-portable.zip`

也就是说：

- **普通用户**：拿成品包，解压或双击直接用
- **不要**让小白面对源码仓库、脚本、依赖说明
- **不要**让小白自己看一堆 Shell/Python 文件

如果你是项目维护者，要打包成品给别人，请看 `DEVELOPMENT.md`

---

## 这个项目现在能做到什么

当前版本已经支持：

- **Windows 真正免安装独立版**：`PT-BDtool.exe`
- **macOS 真正免安装独立版**：`PT-BDtool.app`
- **Windows 便携配置**：优先保存在 `exe` 同目录
- **macOS 便携配置**：优先保存在 `.app` 同级目录
- **VPS 自动依赖检测**：优先支持 `Debian` / `Ubuntu` / `Alpine`
- **完整主流程**：扫描 → 选择条目 → 生成 → 下载到本机 → 清理 VPS 输出

---

## 你要准备什么

开始前，你只需要准备 4 样东西：

- 一台 VPS
- VPS 地址
- VPS 端口
- VPS 密码

另外，VPS 上要先有你想处理的文件，例如：

- 视频文件
- 音频文件
- `BDMV` 目录
- `ISO` 文件

推荐的 VPS 系统：

- `Debian`
- `Ubuntu`
- `Alpine`

## 小白怎么用

### 第一步：打开程序

- Windows：双击 `PT-BDtool.exe`
- macOS：双击 `PT-BDtool.app`

如果 macOS 第一次拦住你，就右键应用，点一次“打开”。

### 第二步：填信息

打开后，填这几项：

- VPS 地址
- SSH 端口
- SSH 密码
- 本机保存目录

如果你不知道“扫描白名单”是什么，**直接留空**。

### 第三步：扫描

点：

```text
扫描 VPS 候选
```

程序会自动去 VPS 上找可处理的文件。

### 第四步：双击条目

扫出来以后，**双击你要处理的条目**。

后面程序会自动做这些事：

1. 在 VPS 上生成截图和媒体信息
2. 打包结果
3. 下载到你的电脑
4. 清理 VPS 上这次生成的输出目录

---

## 处理完成后你会得到什么

常见输出大概是这样：

### 视频

- `mediainfo.txt`
- `1.png` 到 `6.png`

### 音频

- `mediainfo.txt`
- `频谱图.png`

### 原盘 / ISO

- `BDInfo.txt`
- `1.png` 到 `6.png`

---

## 结果会下载到哪里

会下载到你在程序里填的：

```text
本机保存目录
```

如果你没改，一般就是桌面或下载目录。

---

## VPS 会不会自动清理

默认会清理：

- **会清理**：这次生成出来的输出目录
- **不会清理**：你原本放在 VPS 上的源文件

所以正常情况下，不会误删你原始视频。

## 常见问题

### 1）双击没反应

先确认你拿到的是：

- `PT-BDtool.exe`
- 或 `PT-BDtool.app`

**不要让普通用户下载仓库源码 ZIP。**

### 2）Windows 被系统拦住

这通常是 SmartScreen。

按系统提示点：

```text
更多信息 → 仍要运行
```

### 3）macOS 被系统拦住

第一次右键应用，点：

```text
打开
```

之后一般就正常了。

### 4）扫不到文件

先确认你的文件真的在 VPS 上。  
如果你不懂“扫描白名单”，就保持留空，不要乱填。

### 5）VPS 太老，跑不动

最稳的选择还是：

- Debian
- Ubuntu
- Alpine

如果是特别老的系统，可能依赖装不上，或者运行包不兼容。

### 6）能不能保证任何空白 VPS 都 100% 免配置

**不能保证 100%。**

现在能做到的是：

- 优先自动检测系统
- 优先自动安装依赖
- 不行再尝试回退运行包
- 如果还是不行，会明确报错，不再静默失败
